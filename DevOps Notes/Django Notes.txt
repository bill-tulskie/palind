
To restart django
# python manage.py runserver

The backend folder contains a prevalence folder and a datasets folder.  Each has a migrations folder.

from (venv) # prompt while in (cd'd to) backend directory 

To run migrations on a specific app e.g. prevalence or datasets:
(venv) # python manage.py migrate prevalence (or ...datasets)

To run all migrations:
(venv) # python manage.py migrate

Example of truncating corrupted tables:

First start the Django shell

# cd backend
# source venv/bin/activate (bash)
 or 
# . venv/bin/activate (posix)

(venv) # python manage.py shell 

In the Django shell:

>>> from django.db import connection

>>> with connection.cursor() as cursor:
        # Delete all rows from the problematic table
        cursor.execute("DELETE FROM prevalence_patientsbysource")
        cursor.execute("DELETE FROM prevalence_globalstats")
